MEMORIA TÉCNICA - PROYECTO KILOMBOCRM
=====================================

INFORMACIÓN DEL PROYECTO
------------------------
- Nombre: KilomboCRM
- Versión: 1.0.0
- Fecha: Diciembre 2024
- Lenguaje: Java 17
- Framework: Swing (GUI), JDBC (BD)
- Base de Datos: MySQL 8.0+
- Arquitectura: Clean Architecture

OBJETIVOS DEL PROYECTO
----------------------
Desarrollar un sistema de gestión de clientes y pedidos con interfaz gráfica que permita:
- Gestión completa de clientes (CRUD)
- Gestión de pedidos asociados a clientes
- Interfaz visual intuitiva con Java Swing
- Conexión a base de datos MySQL mediante JDBC
- Validación de datos y manejo de excepciones
- Funcionalidades adicionales (BI, WhatsApp, Email)

ESTRUCTURA DEL PROYECTO
-----------------------
El proyecto sigue una arquitectura limpia (Clean Architecture) organizada en capas:

1. CAPA DE DOMINIO (Domain)
   - Entidades: Cliente, Pedido, DetallePedido
   - Interfaces de repositorio
   - Excepciones de dominio
   - Reglas de negocio puras

2. CAPA DE APLICACIÓN (Application)
   - Servicios de negocio (ClienteService, PedidoService)
   - DTOs para transferencia de datos
   - Coordinación de casos de uso

3. CAPA DE INFRAESTRUCTURA (Infrastructure)
   - Implementaciones de repositorios (JDBC)
   - Conexión a base de datos (ConexionBD - Singleton)
   - Mappers para conversión DTO/Entity
   - Servicios externos (WhatsApp, Email)

4. CAPA DE PRESENTACIÓN (Presentation)
   - Interfaz gráfica Swing (MainFrame, Paneles)
   - Modelos de tabla
   - Diálogos modales
   - Navegación entre módulos

BASE DE DATOS
-------------
- Nombre: kilombo (configurable)
- Tablas principales:
  * clientes (id, nombre, apellido, email, telefono)
  * pedidos (id, id_cliente, fecha, total, estado)
  * detalles_pedido (id, id_pedido, tipo_producto, descripcion, cantidad, costo_unitario, precio_unitario)

- Características:
  * Claves foráneas con CASCADE
  * Índices para optimización
  * Constraints de integridad
  * Codificación UTF-8

TECNOLOGÍAS UTILIZADAS
----------------------
- Java 17 (OpenJDK)
- MySQL Connector/J 8.0.33

PATRONES DE DISEÑO IMPLEMENTADOS
---------------------------------
- Singleton: ConexionBD (gestión única de conexión)
- Repository: Abstracción de acceso a datos
- Service Layer: Coordinación de lógica de negocio
- DTO: Transferencia de datos entre capas
- Factory: Creación de servicios y repositorios
- Observer: Eventos en interfaz gráfica
- Template Method: Preparado para repositorios

FUNCIONALIDADES IMPLEMENTADAS
-----------------------------
1. GESTIÓN DE CLIENTES:
   - Crear, leer, actualizar, eliminar clientes
   - Validación de email único
   - Búsqueda por nombre/apellido/email
   - Ver pedidos de un cliente

2. GESTIÓN DE PEDIDOS:
   - Crear pedidos asociados a clientes
   - Estados: PENDIENTE, EN_PROCESO, COMPLETADO, CANCELADO
   - Cálculo automático de totales
   - Filtrado por cliente
   - Detalles de productos por pedido

3. INTERFAZ DE USUARIO:
   - Navegación por pestañas
   - Tablas interactivas con ordenamiento
   - Formularios de creación/edición
   - Diálogos de confirmación
   - Manejo de errores visual

4. FUNCIONALIDADES ADICIONALES:
   - Dashboard BI con estadísticas
   - Envío de mensajes WhatsApp
   - Sistema de emails (con problema conocido)
   - Configuración dinámica de BD
   - Exportación de datos (preparado)

COMPILACIÓN Y EJECUCIÓN
-----------------------
El proyecto ofrece múltiples formas de ejecución según el perfil del usuario:

FORMA 1: EJECUCIÓN COMO USUARIO FINAL (RECOMENDADA)
--------------------------------------------------
1. Requisitos: Java 17+ instalado
2. Archivo: KilomboCRM-1.0.0-jar-with-dependencies.jar
3. Ejecución: java -jar KilomboCRM-1.0.0-jar-with-dependencies.jar
4. Ventajas: No requiere compilar, portable, todas las dependencias incluidas
5. Configuración BD: Desde la interfaz de la aplicación (panel Configuración)

FORMA 2: EJECUCIÓN DESDE CÓDIGO FUENTE (DESARROLLADORES)
-------------------------------------------------------
1. Requisitos: Java 17, MySQL 8.0+, Maven 3.6+
2. Configuración: application.properties (credenciales BD)
3. Compilación: mvn clean compile
4. Ejecución: mvn exec:java -Dexec.mainClass="com.kilombo.crm.presentation.MainFrame"
5. Ventajas: Código fuente disponible, personalizable

FORMA 3: JAR GENERADO DESDE FUENTE (EQUIPOS DE DESARROLLO)
---------------------------------------------------------
1. Requisitos: Java 17, MySQL 8.0+, Maven 3.6+
2. Generación: mvn clean package
3. Archivo generado: target/KilomboCRM-1.0.0-jar-with-dependencies.jar
4. Ejecución: java -jar target/KilomboCRM-1.0.0-jar-with-dependencies.jar
5. Ventajas: Combina lo mejor de ambos mundos
